# CMS Getting Started

## First Time Access

### Login Process

1. Navigate to `/admin` on the website
2. Click "Login with GitHub"
3. Authorize the application
4. Complete GitHub 2FA if enabled
5. You'll be redirected back to the CMS

### Authentication

- **Provider**: GitHub
- **OAuth**: Self-hosted OAuth server
- **2FA Support**: Yes, GitHub 2FA works
- **Permissions**: Read/write to content repository

## CMS Interface

### Main Dashboard
- Content collections list
- Recent changes
- Quick actions
- Media library access

### Navigation
- **Content**: All content collections
- **Media**: Image and file management
- **Workflow**: Editorial workflow (if enabled)

## Content Collections

### Available Collections
1. **Blog** - Blog posts and articles
2. **Services** - Service descriptions
3. **Portfolio** - Case studies
4. **Team** - Team members
5. **Testimonials** - Client testimonials
6. **Landing Pages** - Landing page templates
7. **Lead Magnets** - Lead magnet content
8. **Reviews** - Client reviews
9. **Settings** - Site settings

## Creating Your First Entry

1. Select a collection (e.g., "Blog")
2. Click "New [Collection Name]"
3. Fill in required fields
4. Add optional content
5. Upload images if needed
6. Click "Publish" or "Save as Draft"

## Editing Existing Content

1. Select collection
2. Find entry in list
3. Click to open
4. Make changes
5. Save or publish

## Best Practices

1. **Use Descriptive Titles**: Clear, SEO-friendly titles
2. **Add Images**: Visual content performs better
3. **Save Regularly**: Don't lose work
4. **Preview Before Publishing**: Check how it looks
5. **Follow Guidelines**: Maintain consistency
