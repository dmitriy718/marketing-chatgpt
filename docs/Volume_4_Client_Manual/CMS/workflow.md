# CMS Workflow

## Editorial Workflow

### Draft → Review → Publish

1. **Create Draft**
   - Create new entry
   - Save as draft
   - Work on content

2. **Review**
   - Preview content
   - Check formatting
   - Verify links and images

3. **Publish**
   - Click "Publish"
   - Content goes live
   - Git commit created

## Git Workflow

### Automatic Commits
- Each save creates a Git commit
- Commit message auto-generated
- Changes visible in Git history

### Branch Strategy
- All changes go to main branch
- No branching in CMS
- Direct commits to production

## Collaboration

### Multiple Editors
- Multiple users can edit
- Last save wins (no conflict resolution)
- Coordinate via other channels

### Best Practices
- Communicate before editing
- Don't edit same entry simultaneously
- Review changes in Git if needed

## Content Review Process

1. **Create**: Author creates content
2. **Draft**: Save as draft
3. **Review**: Preview and check
4. **Publish**: Make live
5. **Monitor**: Check for issues

## Version Control

### Git History
- All changes tracked in Git
- Can view history
- Can revert if needed

### Rollback
- View Git history
- Find previous version
- Restore if needed
